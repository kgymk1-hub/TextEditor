Pocket Text Editor

Pocket Text Editor は、チャットAIとやり取りしながらWebアプリのコードを編集・確認する作業を支援する、ローカル完結型のPWAテキストエディタです。

AndroidスマホとPCの両方で使えることを目指しており、複数タブ編集、ファイル読み書き、HTMLプレビュー、CSVプレビュー、ZIP操作、自動バックアップ、現在タブコピー、全タブ結合コピー、コーディング用記号バーなどを備えています。

---

特徴

- チャットAIへコードを渡しやすい「現在タブコピー」「全タブ結合コピー」
- スマホで入力しにくい記号を補助するコーディング用記号バー
- 複数ファイルを同時に扱えるタブ編集
- HTMLプレビュー、CSV表形式プレビューに対応
- HTML / CSS / JavaScript を組み合わせて確認できる連携プレビュー
- ZIPファイル内のテキスト系ファイルを開く機能
- 開いている複数タブをまとめてZIP保存する機能
- UTF-8 / UTF-8 BOM / Shift_JIS に対応
- 自動バックアップによる編集内容の復元
- ダークモード対応
- PWA対応

使い方

起動

"index.html" をブラウザで開いて使用します。

PWA対応ブラウザでは、ホーム画面に追加してアプリのように起動できます。

---

ファイル操作

新規

空のタブを作成します。

開く

端末内のテキストファイルを開きます。

対応を想定している主なファイル形式は以下です。

- ".txt"
- ".csv"
- ".html"
- ".htm"
- ".js"
- ".css"
- ".json"
- ".md"

保存

保存先を保持している場合は、同じファイルへ上書き保存します。

保存先を保持していない場合は、ブラウザのダウンロード機能を使って保存します。

DL保存

現在のタブ内容を、ブラウザのダウンロード機能で保存します。

DL保存は保存先を変更しません。

現在タブコピー

現在開いているタブの本文をクリップボードへコピーします。

チャットAIへ現在のコードを渡したい時に使います。

全タブ結合コピー

開いている全タブの内容を、ファイル名付きで1つのテキストにまとめてクリップボードへコピーします。

出力イメージ：

===== index.html =====
...

===== css/style.css =====
...

===== js/app.js =====
...

チャットAIに複数ファイルの状態をまとめて渡したい時に使います。

---

保存について

このアプリの保存機能は、利用環境によって挙動が異なります。

File System Access API が使える環境

File System Access API が使える環境では、「開く」で開いた既存ファイルに対して上書き保存できます。

File System Access API が使えない環境

File System Access API が使えない環境では、保存はブラウザのダウンロード機能を使います。

この場合、アプリ側は保存先フォルダを保持できません。

文字コードについて

読み込み時の文字コードを選択できます。

- UTF-8
- Shift_JIS

保存時の文字コードを選択できます。

- UTF-8
- UTF-8 BOM付き

プレビュー機能

HTMLプレビュー

現在のタブ内容をHTMLとしてプレビューします。

連携プレビュー

連携プレビューをONにすると、開いているCSSタブとJavaScriptタブの内容をHTMLプレビューへ組み込んで確認できます。

例えば、以下のような複数タブを開いている場合に便利です。

- "index.html"
- "style.css"
- "app.js"

CSVプレビュー

現在のタブ内容をCSVとして表形式で表示します。

CSVの1行目をヘッダーとして扱うかどうかは、設定から切り替えできます。

---

ZIP機能

ZIP開く

ZIPファイルを開き、ZIP内のテキスト系ファイル一覧を表示します。

ZIP内ファイルを開く

ZIP内のファイルを個別にタブとして開けます。

一括展開

ZIP内の対応ファイルをまとめてタブとして開けます。

ZIP保存

現在開いている全タブを、1つのZIPファイルとして保存します。

設定

設定パネルから以下を変更できます。

- 読み込み文字コード
- 保存文字コード
- CSV 1行目ヘッダー
- 連携プレビュー
- 文字サイズ
- 折り返し

記号バー表示

スマホでコードを書く時に使いやすいように、よく使う記号をワンタップで入力できる記号バーを用意しています。

表示方法は以下から選べます。

- 常に表示
- 編集中のみ表示
- 非表示

---

自動バックアップ

編集中のタブ内容は、ブラウザの "localStorage" に自動バックアップされます。

前回のバックアップが残っている場合、起動時に復元確認が表示されます。

注意

大きなファイルを多数開いた場合、ブラウザの保存容量制限により自動バックアップに失敗する可能性があります。

各ファイルの役割

"index.html"

画面構造を定義しています。

"css/style.css"

レイアウト、色、レスポンシブ表示、ダークモードなどを定義しています。

"js/app.js"

アプリ全体の初期化、DOM取得、イベント登録、画面状態管理を担当します。

"js/tabs.js"

タブの作成、切り替え、削除、未保存状態の管理を担当します。

"js/editor.js"

エディタ入力、検索、置換、すべて置換、記号挿入を担当します。

"js/file-service.js"

ファイルの読み込み、保存、DL保存、ZIP操作を担当します。

"js/preview-service.js"

HTMLプレビュー、連携プレビュー、CSVプレビューを担当します。

"js/storage-service.js"

自動バックアップ、テーマ設定、文字コード設定、アプリ設定の保存と読み込みを担当します。

Android判定、File System Access API判定、Service Worker登録可否など、実行環境の判定を担当します。

"service-worker.js"

PWA用のキャッシュ処理を担当します。

"manifest.json"

PWAとしてのアプリ名、表示方式、アイコンなどを定義しています。

開発方針

このアプリは、HTML / CSS / Vanilla JavaScript で作成しています。

React、Vue、Svelteなどのフレームワークは使用していません。

Androidスマホでも扱いやすいように、以下を重視しています。

- 画面を広く使えるUI
- タップ操作しやすいボタン
- チャットAIへコードを渡しやすいコピー機能
- ローカルファイル中心のシンプルな構成
- PWAとして動作できること
- 将来的なAndroidアプリ化を見据えた構成

---

注意事項

- 自動バックアップには "localStorage" を使用しています。
- 大きなファイルを多数開くと、ブラウザの保存容量制限に達する可能性があります。
- DL保存はブラウザのダウンロード機能を利用します。
- DL保存では、アプリ側で保存先フォルダを保持できません。
- HTML連携プレビューでは、設定ON時のみJavaScript実行を許可します。
- 信頼できないHTMLやJavaScriptの実行には注意してください。
- ブラウザや端末によって、File System Access APIの対応状況が異なります。
